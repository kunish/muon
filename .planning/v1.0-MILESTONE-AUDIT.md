## Cross-Phase Integration Findings

### 1. Search surface split-brain blocks milestone retrieval completion

- `ChatHeader.vue` opens the `search` side panel.
- `ChatWindow.vue` mounts `SearchMessages.vue` for that panel.
- `GlobalSearch.vue` and `retrievalStore.ts` contain the milestone's cross-conversation retrieval, pagination, virtualization, and bounded preload logic.

Result: the milestone's most advanced retrieval implementation exists but is **not production-reachable**.

### 2. Reachable search result jump path is not connected

- `SearchMessages.vue` emits `jumpTo`
- `ChatWindow.vue` does not consume it

Result: the real, user-reachable search side panel can still dead-end before navigation.

### 3. Offline digest reopen path breaks historical continuity

- `OfflineDigestPanel.vue` immediately calls `digestStore.initializeDigest()`
- `initializeDigest()` hydrates saved entries, then calls `buildDigestSession()`
- `buildDigestSession()` overwrites `entries.value` with the new session, including the empty-session case

Result: saved digest history can disappear on reopen.

### 4. Digest still has no away-window backfill source

- `useNetworkStatus` tracks `lastOfflineAt`
- `digestStore` only listens to `matrixEvents.on('room.message')` after mount
- no backfill path scans already-synced messages between `lastOfflineAt` and now

Result: away-window digest generation remains unreliable, which also weakens digest-fed decision suggestions.
